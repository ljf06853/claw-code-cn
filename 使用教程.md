## 1. 环境准备

### 必需工具

| 工具      | 最低版本   | 说明                             |
|-----------|-----------|----------------------------------|
| **Rust**  | 1.77+     | 需要支持 edition 2021 及 Rust 2024 特性 |
| **Cargo** | 随 Rust    | Rust 包管理器                      |
| **Git**   | 2.x       | 代码版本控制                        |

### 安装 Rust

```powershell
# Windows (PowerShell)
winget install Rustlang.Rust.MSVC

# 或使用 rustup
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

## 2. 编译项目

```powershell
cd D:\newworkspace\claw-code-main\rust

# 开发模式编译（更快，带调试信息）
cargo build --bin claw

# Release 模式编译（优化后的可执行文件）
cargo build --bin claw --release
```

编译产物路径：
- 开发模式：`target/debug/claw.exe`
- Release 模式：`target/release/claw.exe`

## 3. 模型配置

Claw Code 支持多种 AI 模型 Provider。通过设置对应的**环境变量**即可切换不同的模型。

### 环境变量优先级规则

```
CLAW_API_KEY（通用自定义）优先级最高
     ↓
根据模型名前缀自动匹配 Provider
     ↓
按顺序检测各 Provider 的环境变量
     ↓
回退到 Anthropic Claude
```

> **重要**：如果同时设置了 `CLAW_API_KEY` 和其他 Provider 的环境变量，`CLAW_API_KEY` 会优先生效。

---

### 3.1 Anthropic Claude（默认）

这是默认的 Provider，无需额外配置（需要 Anthropic 凭据）。

| 环境变量                | 说明                    | 必填 |
|------------------------|------------------------|------|
| `ANTHROPIC_API_KEY`    | Anthropic API 密钥     | ✅   |
| `ANTHROPIC_AUTH_TOKEN` | OAuth Bearer Token     | 可选 |
| `ANTHROPIC_BASE_URL`   | 自定义 API 地址         | 可选 |

**默认 API 地址**：`https://api.anthropic.com`

**支持的模型名 / 别名**：

| 模型别名   | 实际模型名                        |
|-----------|----------------------------------|
| `opus`    | `claude-opus-4-6`                |
| `sonnet`  | `claude-sonnet-4-6`              |
| `haiku`   | `claude-haiku-4-5-20251213`      |

**示例**：

```powershell
$env:ANTHROPIC_API_KEY = "sk-ant-xxxxx"

# 使用默认模型 (claude-opus-4-6)
cargo run --bin claw

# 使用 Sonnet 模型
cargo run --bin claw -- --model sonnet
```

也可以使用 OAuth 登录：

```powershell
cargo run --bin claw -- login
```

### 3.10 通用自定义 Provider

如果你使用的模型不在上面的列表中，或者你通过第三方中转/代理使用模型，可以使用**通用自定义 Provider**。

| 环境变量          | 说明                  | 必填 | 默认值 |
|------------------|----------------------|----|------|
| `CLAW_API_KEY`   | 自定义 Provider 的密钥  | ✅  | -    |
| `CLAW_BASE_URL`  | 自定义 API 地址         | ✅  | `https://api.openai.com/v1` |

> ⚠️ **注意**：`CLAW_API_KEY` 的优先级最高。当该变量存在时，所有模型都会通过自定义 Provider 发送请求，忽略其他 Provider 的环境变量。

**示例 — 使用 MiniMax 中转**：

```powershell
$env:CLAW_API_KEY = "sk-your-api-key"
$env:CLAW_BASE_URL = "https://api.minimaxi.com/v1"

cargo run --bin claw -- --model MiniMax-M2.7
```

**示例 — 使用本地 Ollama**：

```powershell
$env:CLAW_API_KEY = "ollama"
$env:CLAW_BASE_URL = "http://localhost:11434/v1"

cargo run --bin claw -- --model llama3
```

**示例 — 使用 OpenRouter**：

```powershell
$env:CLAW_API_KEY = "sk-or-xxxxx"
$env:CLAW_BASE_URL = "https://openrouter.ai/api/v1"

cargo run --bin claw -- --model anthropic/claude-3.5-sonnet
```

## 4. 启动方式

### 4.1 交互模式（REPL）

最常用的方式——直接启动，进入交互式对话界面：

```powershell
# 直接启动（使用默认模型）
cargo run --bin claw

# 指定模型
cargo run --bin claw -- --model sonnet

# 如果已编译，直接运行可执行文件
.\target\debug\claw.exe --model deepseek-chat
```

### 4.2 单次提问模式

不进入交互界面，执行一次提问后立即退出：

```powershell
# 方式一：直接在命令后面写提示词
cargo run --bin claw -- "总结一下这个仓库的功能"

# 方式二：使用 -p 参数
cargo run --bin claw -- -p "解释 src/main.rs 的作用"

# 方式三：使用 prompt 子命令
cargo run --bin claw -- prompt "列出所有 TODO"
```

### 4.3 管道模式

可以通过管道将输入传递给 Claw：

```powershell
# 将文件内容作为上下文
Get-Content .\README.md | cargo run --bin claw -- "总结这份文档"

# 将命令输出作为上下文
git diff | cargo run --bin claw -- "帮我写 commit message"
```

## 5. 命令行参数详解

### 主要参数

| 参数                               | 简写  | 默认值                     | 说明 |
|-----------------------------------|------|--------------------------|------|
| `--model MODEL`                   |      | `claude-opus-4-6`        | 指定使用的 AI 模型名称，支持别名 |
| `--output-format FORMAT`          |      | `text`                   | 输出格式：`text`、`json`、`ndjson` |
| `--permission-mode MODE`          |      | `danger-full-access`     | 权限模式 |
| `--dangerously-skip-permissions`  |      |                          | 跳过所有权限检查 |
| `--allowedTools TOOLS`            |      |                          | 限制可用工具（可重复；支持逗号分隔） |
| `--version`                       | `-V` |                          | 打印版本和构建信息 |
| `--help`                          | `-h` |                          | 打印帮助信息 |

---

### `--model` 参数详解

可以使用模型的完整名称，也可以使用简短别名：

```powershell
# 别名
cargo run --bin claw -- --model opus       # → claude-opus-4-6
cargo run --bin claw -- --model sonnet     # → claude-sonnet-4-6
cargo run --bin claw -- --model haiku      # → claude-haiku-4-5-20251213
cargo run --bin claw -- --model grok       # → grok-3
cargo run --bin claw -- --model grok-mini  # → grok-3-mini

# 完整模型名 — 直接传入即可
cargo run --bin claw -- --model deepseek-chat
cargo run --bin claw -- --model MiniMax-M2.7
cargo run --bin claw -- --model qwen-turbo
```

也支持 `=` 号写法：

```powershell
cargo run --bin claw -- --model=sonnet
```

---

### `--permission-mode` 参数详解

控制 Claw 对文件系统和工具的访问权限：

| 值                     | 说明                                |
|-----------------------|------------------------------------|
| `read-only`           | 只读模式，只能读取文件和搜索            |
| `workspace-write`     | 工作区写入模式，允许修改当前工作区的文件   |
| `danger-full-access`  | 完全访问模式（默认），可以执行任意操作     |

```powershell
# 只读模式 — 安全，只让 AI 看代码
cargo run --bin claw -- --permission-mode read-only

# 工作区写入 — 允许 AI 修改文件但不能执行命令
cargo run --bin claw -- --permission-mode workspace-write
```

---

### `--output-format` 参数详解

控制非交互模式下的输出格式：

| 值      | 说明                                    |
|--------|----------------------------------------|
| `text` | 纯文本输出（默认），适合人类阅读             |
| `json` | JSON 格式输出，适合程序解析                |
| `ndjson`| 换行分隔的 JSON，适合流式处理              |

```powershell
# JSON 输出 — 方便集成到脚本
cargo run --bin claw -- --output-format json prompt "解释 main.rs"
```

---

### `--allowedTools` 参数详解

限制 AI 可以使用的工具集合：

```powershell
# 只允许读取和搜索
cargo run --bin claw -- --allowedTools read,glob "总结 Cargo.toml"

# 多次指定
cargo run --bin claw -- --allowedTools read --allowedTools bash "运行测试"
```

## 6. 子命令说明

```powershell
cargo run --bin claw -- skills
```

## 7. REPL 交互命令

在交互模式下，可以使用以 `/` 开头的斜杠命令：

| 命令                      | 说明                        |
|--------------------------|----------------------------|
| `/help`                  | 显示完整的斜杠命令列表         |
| `/model <name>`          | 在会话中切换模型              |
| `/status`                | 查看当前会话和工作区状态        |
| `/permissions <mode>`    | 调整工具访问权限              |
| `/vim`                   | 切换 Vim 编辑模式            |
| `/agents`                | 列出已配置的 Agent            |
| `/skills`                | 列出已安装的 Skill            |
| `Tab`                    | 自动补全斜杠命令              |

**常用 REPL 操作**：

```
> 帮我写一个 hello world 程序       ← 直接输入提示词
> /model sonnet                      ← 切换到 Sonnet 模型
> /status                            ← 查看当前状态
> /help                              ← 获取帮助
```

## 8. 常见问题

### Q1: 报错 `missing Claw credentials; export ANTHROPIC_AUTH_TOKEN or ANTHROPIC_API_KEY`

**原因**：没有设置任何 Provider 的 API Key。

**解决方案**：根据你使用的模型，设置对应的环境变量。例如使用 DeepSeek：

```powershell
$env:DEEPSEEK_API_KEY = "your-api-key"
cargo run --bin claw -- --model deepseek-chat
```

---

### Q2: 如何在 Windows 上持久化环境变量？

PowerShell 中 `$env:VAR = "value"` 只在当前会话有效。持久化设置方法：

```powershell
# 方式一：写入 PowerShell Profile（推荐）
notepad $PROFILE
# 在文件中添加下面的行：
# $env:DEEPSEEK_API_KEY = "your-key"

# 方式二：系统环境变量
[Environment]::SetEnvironmentVariable("DEEPSEEK_API_KEY", "your-key", "User")
```

---

### Q3: 如何同时配置多个 Provider 并切换？

你可以同时设置多个 Provider 的环境变量，然后通过 `--model` 参数指定使用哪个：

```powershell
# 同时设置多个 Key
$env:DEEPSEEK_API_KEY = "your-deepseek-key"
$env:QWEN_API_KEY = "your-qwen-key"
$env:MINIMAX_API_KEY = "your-minimax-key"

# 启动时选择
cargo run --bin claw -- --model deepseek-chat     # 使用 DeepSeek
cargo run --bin claw -- --model qwen-turbo        # 使用千问
cargo run --bin claw -- --model MiniMax-M2.7      # 使用 MiniMax
```

> ⚠️ **注意**：不要同时设置 `CLAW_API_KEY`，否则它会覆盖所有其他 Provider。

---

### Q4: 如何使用代理访问？

如果你需要通过代理访问 API，可以设置标准的 HTTP 代理环境变量：

```powershell
$env:HTTPS_PROXY = "http://127.0.0.1:7890"
$env:HTTP_PROXY = "http://127.0.0.1:7890"

cargo run --bin claw -- --model sonnet
```

---

### Q5: 模型名自动识别规则是什么？

Claw Code 按以下规则自动匹配 Provider：

| 模型名前缀                | 匹配的 Provider |
|--------------------------|----------------|
| `claude-*`、`opus`、`sonnet`、`haiku` | Anthropic Claude |
| `grok*`                  | xAI Grok       |
| `minimax*`、`abab*`      | MiniMax        |
| `deepseek*`              | DeepSeek       |
| `qwen*`                  | 千问 (Qwen)     |
| `doubao*`、`ep-*`        | 豆包 (Doubao)   |
| `glm*`                   | 智谱 GLM        |
| `gemini*`                | Gemini         |
| 其他                     | 如设置了 `CLAW_API_KEY` 则走自定义 Provider |
